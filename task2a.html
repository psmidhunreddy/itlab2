Task II: Interactive Website using JavaScript

1. Which website/section of the website did you choose? Why?

I choose the School Time Table section. I selected this because the timetable is very big and confusing when everything is shown together. It is hard to find specific subject, so I thought this section is good for adding interaction.

2. What did you identify as weaknesses in that page?

The page was showing full timetable at once which is too much information. It was difficult to search one course. Also there was no filtering option and no interaction.

3. What are your main ideas for improving the page?

My main idea was to add dropdown options so user can select program or course. Instead of showing everything, only required data will be displayed. This makes it easy to understand.

4. Which interactive elements did you incorporate into your design?

I used dropdown menus with onchange event. I used JavaScript to dynamically generate timetable. I also used event listeners and conditional statements for filtering.

5. Give two specific examples from the page that you think are better after your redesign. Explain why.

First example is when selecting a program, only that program timetable is shown. It reduces confusion. Second example is when selecting a course, only that course time slots are displayed. So student can easily see when that subject is there.

6. Which mouse and keypress elements did you use?

I used mouse change event for dropdown selection. I did not use keypress event, but it can be added for search functionality.

7. How does mouseover event differ from mouseenter and mouseleave events?

mouseover event works when mouse moves over element and also its child elements. mouseenter works only when mouse enters that element one time. mouseleave works when mouse leaves that element.
